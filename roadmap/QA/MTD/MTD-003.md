# MTD-003 — PIX End-to-End com Sandbox Asaas

## Flow
[FLOW-003](../Flow/FLOW-003.md) — PIX End-to-End Recorrente

## Objetivo
Validar o ciclo completo do pagamento PIX recorrente: recebimento do webhook, exibição do QR Code na página de billing, envio do e-mail de vencimento e desaparecimento automático do card após confirmação de pagamento.

## Pré-condições
- Ambiente com Asaas sandbox configurado (`ASAAS_API_KEY` sandbox)
- Assinatura ativa com método PIX criada no sandbox
- Acesso ao painel do Asaas sandbox para disparar webhooks manualmente
- Acesso ao cliente de e-mail do usuário de teste (ou painel do MailerSend sandbox)
- Variável `ASAAS_WEBHOOK_TOKEN` configurada corretamente no ambiente

---

## Cenário 1 — QR Code PIX aparece na página de billing

### Passos
1. Logar como usuário com assinatura PIX ativa no sandbox
2. Acessar `/billing` — confirmar que **não há card** PIX pendente inicialmente
3. No painel sandbox do Asaas, navegar até a cobrança do ciclo atual
4. Disparar manualmente o evento `PAYMENT_CREATED` com `billingType=PIX` via webhook sandbox
5. Aguardar 5–10 segundos para processamento
6. Recarregar a página `/billing`
7. Verificar se o card "Pagamento PIX Pendente" aparece

### Resultado Esperado
- Card exibido com:
  - [ ] Imagem do QR Code (renderizada corretamente, não quebrada)
  - [ ] String "Copia e Cola" visível abaixo do QR Code
  - [ ] Botão "Copiar" funcional (clicar e verificar clipboard)
  - [ ] Countdown de expiração exibindo tempo real baseado em `pixExpiresAt`
  - [ ] Valor da cobrança formatado em R$ (ex: R$ 49,90)

### Evidência
- [ ] Screenshot do card completo com QR Code visível
- [ ] Confirmação de cópia do Copia e Cola (toast ou feedback visual)

---

## Cenário 2 — E-mail de lembrete no PAYMENT_OVERDUE

### Passos
1. Com o card PIX pendente visível (após Cenário 1)
2. No painel sandbox do Asaas, disparar evento `PAYMENT_OVERDUE` para a mesma cobrança
3. Aguardar 10–15 segundos para processamento
4. Verificar a caixa de entrada do e-mail do usuário de teste
5. Verificar o conteúdo do e-mail recebido

### Resultado Esperado
- E-mail recebido com:
  - [ ] Assunto relacionado a cobrança vencida
  - [ ] Valor da cobrança formatado em R$
  - [ ] Data de vencimento
  - [ ] Botão ou link "Pagar com PIX" (apontando para `paymentLink` do Asaas)
  - [ ] Template consistente com os outros e-mails do produto (em português)
- Na página `/billing`: banner vermelho "Sua cobrança PIX venceu" visível no card

### Evidência
- [ ] Screenshot do e-mail recebido com conteúdo completo
- [ ] Screenshot do card com banner OVERDUE em vermelho

---

## Cenário 3 — Card desaparece após pagamento confirmado

### Passos
1. Com o card PIX visível na página `/billing`
2. No painel sandbox do Asaas, confirmar o pagamento (disparar `PAYMENT_RECEIVED`)
3. Permanecer na página `/billing` sem recarregar
4. Aguardar até **30 segundos** (intervalo de auto-refresh do `refetchInterval`)
5. Verificar se o card desaparece automaticamente

### Resultado Esperado
- Card "Pagamento PIX Pendente" desaparece em até 30 segundos
- Sem reload manual necessário
- Página permanece funcional (sem erro ou estado quebrado)

### Evidência
- [ ] Gravação de tela ou sequência de screenshots mostrando card presente → card ausente

---

## Cenário 4 — QR Code expirado

### Passos
1. Criar uma cobrança PIX com `pixExpiresAt` no passado (ou aguardar expiração natural no sandbox)
2. Acessar `/billing` com o pagamento pendente expirado

### Resultado Esperado
- Mensagem "QR Code expirado" exibida em vez do QR Code
- Botão ou link "Solicitar novo QR Code" apontando para o `paymentLink` do Asaas

### Evidência
- [ ] Screenshot do estado de QR Code expirado

---

## Cenário 5 — Usuário com cartão de crédito

### Passos
1. Logar como usuário com assinatura via **cartão de crédito**
2. Disparar evento `PAYMENT_CREATED` com `billingType=CREDIT_CARD` via sandbox
3. Acessar `/billing`

### Resultado Esperado
- Card "Pagamento PIX Pendente" **NÃO** aparece
- Página de billing exibe o estado normal (sem pending PIX)

### Evidência
- [ ] Screenshot da página /billing sem card PIX para usuário de cartão

---

## Cenário 6 — Responsividade mobile

### Passos
1. Com o card PIX visível, acessar DevTools → Toggle Device Toolbar
2. Selecionar viewport mobile (ex: iPhone 12 = 390px, ou 375px)
3. Verificar layout do card

### Resultado Esperado
- Card responsivo, sem overflow horizontal
- QR Code redimensionado para caber na tela
- Botão "Copiar" acessível sem scroll

### Evidência
- [ ] Screenshot do card em viewport mobile

### Notas / Riscos
- Botão "Alterar para Cartão de Crédito" intencionalmente não implementado (TASK-048 retrospectiva) — não reportar como bug
- Se o Asaas sandbox não estiver disponível, verificar se existe mock/stub para teste offline
